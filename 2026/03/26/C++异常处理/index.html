C++异常处理机制

misc | Hexo

C++ 异常处理是一种在程序运行时处理错误和特殊情况的机制。它允许程序在遇到问题时，通过抛出和捕获异常来转移控制流，而不是使用传统的错误处理方法（如通过函数返回值）。C++ 异常处理主要涉及三个关键字：try、catch和throw。

核心工作机制：栈展开 (Stack Unwinding)

这是C++异常机制中最重要的一环。当异常被 throw 抛出，且在当前作用域没有找到匹配的 catch 块时，C++ 运行时会去调用者的作用域中寻找，一直沿着函数调用栈向上找。

在这个“向上找”的过程中，C++ 会自动调用当前作用域内所有已构造的局部对象的析构函数，安全地释放资源。这个过程就叫栈展开。这与 RAII（资源获取即初始化）机制完美配合，防止了内存泄漏。

异常展开机制在把控制权交还给 main 函数时，不仅会设置 RIP，还会从栈上恢复 RBP 寄存器，以恢复 main 函数的栈帧环境。

所以说，如果我们能伪造rbp+8的地址，让它指向一个含有c++异常处理的函数。这样做到可以绕过canary等保护。

由于我们在0x2434，输入key字段的时候，触发了c++异常处理，所以此时会回到0x2434的父函数，也就是0x25b1去处理。

可以看到，我们填的ret函数是0x25db，是因为当我们执行0x2434，也就是去call sub_2434后，下一条指令会被压入栈中，应该是这里的0x25d9，不过由于0x25d9和0x25db都在try{}块中，所以都会被视作是合法的，就会正常进行catch{}块。我们可以看到catch{}块中去判断了抛出错误的类型是不是char类型的。由于抛出的是error key，所以这里会跳转到loc_25f5，然后最后跳转到loc_264a会执行40字节的一些操作(0x18+0x8+0x8=0x28)。所以我们会填入p64(0xdeadbeef)*5，这样等恢复完寄存器，执行ret步骤的时候就刚刚好是我们的rop链的起始地址了。
