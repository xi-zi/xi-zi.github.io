异架构汇编

异架构汇编存在三种，分别是PowerPC，ARM&AARCH64，MIPS

从 x86 到 RISC

x86 是 CISC（复杂指令集），而 ARM、MIPS、PowerPC 均属于 RISC（精简指令集）。

  1. Load/Store 架构：RISC 架构不能直接对内存中的数据进行加减乘除等运算。必须先将数据 Load 到寄存器中，在寄存器里完成运算，然后再 Store 回内存。
  2. 寄存器数量多：通常有 32 个以上的通用寄存器。因此，函数传参主要依赖寄存器，而不是像早期 x86 那样依赖栈。
  3. 指令定长：大多数情况下指令长度固定为 4 字节（除了 ARM 的 Thumb 模式），这对 ROP 链的构造和 shellcode 的编写影响巨大。

PowerPC(工业控制与汽车电子)

PowerPC 架构下的函数调用约定是通过寄存器传递参数，而不是通过栈传递参数。但当参数数量过多时，会将多余的参数压入栈中，因此栈帧中也会包含函数参数。

寄存器设计复杂：拥有 32 个通用寄存器 (GPR)，还有专门的链接寄存器 (LR)、计数器寄存器 (CTR) 和条件寄存器 (CR)。

栈帧结构：PowerPC 没有专门的帧指针寄存器，且它的栈不仅用于保存局部变量，还会在函数入口分配极大的空间，用来保存调用者的环境状态。

指令晦涩：指令助记符繁多且复杂（例如各种条件跳转 b, bl, bctr, blr 等），初期阅读难度较大。

常用寄存器

寄存器 说明
R1/SP 栈顶指针。
R3 存储函数调用的第一个参数或函数的返回值。
R4-R10 函数或系统调用的参数，部分情况下R4寄存器也会作为函数的返回值使用。
LR 链接寄存器，用于存放函数调用结束处的返回地址。
PC 程序计数器。

常用指令

PowerPC 架构的指令通常使用缩写的单词组合来表示其功能

  • **STB：**Store Byte（存储字节）
  • **STW：**Store Word（存储字）
  • LD：Load（加载，将数据从内存中读取到寄存器中）
  • **LI：**Load Immediate（加载立即数）
  • **LIS：**Load Immediate and Shift（装载一个16位立即数到目标寄存器，并将它左移16位，用零填充低位）
  • **LWZ：**Load Word and Zero（从内存读取一个字单元到寄存器中，并将高位清零）
  • **BL：**Branch and Link（当前程序返回地址存入寄存器 LR 并跳转）
  • **MTCTR：**Move to Count Register（将一个32位寄存器的值存储到寄存器 CTR）
  • **BCTRL：**Branch to Count Register and Link（用于调用函数，将当前程序返回地址存入寄存器 LR 并跳去执行寄存器 CTR 指向的指令）
  • @符号 通常用于表示地址的高位和低位。

PowerPC (PPC) 的栈帧设计最为特殊，它强制要求严格的栈回溯链（Back Chain）。

  • 关键寄存器：R1 (栈指针 SP)。PPC 通常没有专门的帧指针。LR 是特殊寄存器，不能直接运算。

  • 栈帧布局（从高地址到低地址）：

    1. 参数保存区（可选）。
    2. 局部变量与寄存器保存区。
    3. LR 保存位置：PPC 函数的返回地址（LR）不保存在自己的栈帧里，而是保存在调用者（Caller）的栈帧里（通常位于 SP + 4 或 SP + 8 的位置）。
    4. Back Chain (栈顶)：当前栈指针 SP (R1) 指向的内存位置 0(R1)，永远存放着上一个栈帧的 SP 指针。这使得 PPC 可以形成一个完美的链表，极其方便调试器进行回溯。

MIPS(传统路由器的统治者)

TP-Link、Netgear 等路由器固件中极为常见，在网络设备领域占据主导。

  • 分支延迟槽 (Branch Delay Slot)：这是 MIPS 最反直觉的特性。在跳转指令（如 j、jal）后面的第一条指令，会在跳转真正发生前执行。阅读汇编时必须习惯将跳转和它的下一条指令绑在一起看。
  • 大小端问题：MIPS 有大端序 (mips) 和小端序 (mipsel)，这是需要特别注意的坑点。
  • 调用约定：前 4 个参数用 $a0-$a3 传递，返回值存在 $v0，返回地址存在 $ra。
  • 缓存一致性 (Cache Coherency)：由于指令缓存和数据缓存分离，自己写入的 shellcode 有时需要显式刷新指令缓存才能被 CPU 执行（通常通过特定的系统调用）。

指令集学习

各寄存器作用如下

REGISTER NAME USAGE
$0 $zero 存储常量 0 (constant value 0)
$1 $at 保留给汇编器 (Reserved for assembler)
$2-$3 $v0-$v1 存放函数调用返回值或表达式 (values for results and expression evaluation)
$4-$7 $a0-$a3 作为函数调用的前四个参数 (arguments)
$8-$15 $t0-$t7 供汇编程序使用的临时寄存器 临时变量
$16-$23 $s0-$s7 调用子函数时 用于保存原寄存器的值 (saved)
$24-$25 $t8-$t9 供汇编程序使用的临时寄存器 补充t0−t7
$26-$27 k0−k1 中断/异常处理程序使用 保存系统参数
$28 $gp 全局指针 (Global Pointer)
$29 $sp 堆栈指针 (Stack Pointer)
$30 $fp 帧指针 (Frame Pointer)
$31 $ra 返回地址 (return address)

除此以外还有 3 个特殊的寄存器，分别是 PC (程序计数器)、HI (乘除结果高位寄存器)、LO (乘除结果低位寄存器)

ARM / AArch64 (最主流的 IoT 架构)

广泛应用于智能家居设备、移动端、树莓派等。

  • 执行状态与指令集：
    • 包含 ARM 模式 (32位定长指令) 和 Thumb 模式 (16位/32位变长指令)。常利用这两者的切换来绕过坏字符或复用 ROP gadget。
    • AArch64 (64位)：寄存器名称从 R0-R15 变成了 X0-X30。
  • 函数调用约定 (ATPCS)：前 4 个参数用 R0-R3 传递（AArch64 是 X0-X7），多余的入栈。返回值存放在 R0。
  • 关键寄存器：
    • R11 (FP)：帧指针，类似 ebp。
    • R13 (SP)：栈指针，类似 esp。
    • R14 (LR)：链接寄存器。函数调用时，返回地址存放在 LR，而不是直接压入栈中。控制程序执行流时，覆盖保存的 LR 值是关键。
    • R15 (PC)：程序计数器，类似 eip。

常用指令

  • MOV： 数据传输指令，用于将数据从一个寄存器或内存中复制到另一个寄存器或内存中。
  • ADD/SUB： 加法和减法指令。
  • CMP： 比较指令，比较两个操作数的大小关系。
  • B： 分支指令，用于无条件跳转到指定地址。
  • BL： 分支指令，用于跳转到指定地址并将返回地址保存在LR寄存器中，可用于函数调用。
  • LDR/STR： 数据传输指令，用于从内存中读取数据或将数据写入内存。
    • LDR 用于将某些内容从内存加载到寄存器中，例如 LDR R2, [R0] 从R0寄存器中存储的内存地址的值读入R2寄存器。
    • STR 用于将某些内容从寄存器存储到内存地址中，例如 STR R2, [R1] 从R2寄存器中将值存储到R1寄存器中的内存地址中。
  • PUSH/POP： 栈操作指令，用于将寄存器中的数据入栈或从栈中弹出数据。

  • 叶子函数（Leaf Function）：函数体内没有调用（call/bl）任何其他函数的函数。它是调用链的末端。

  • 非叶子函数（Non-leaf Function）：函数体内有调用其他函数的函数。它需要保存和恢复自身的返回地址。

  • 非叶子函数（劫持稳定且直接）：由于它必然将返回地址（x86的EIP或ARM的LR）保存在栈上的固定位置，栈溢出时，覆盖该保存值即可在函数退出时精准劫持。攻击偏移量容易计算（通常在 ebp+4 或栈顶下方固定偏移）。

  • 叶子函数（劫持危险性高但脆弱）：在 ARM 中，叶子函数返回时直接从 LR 寄存器跳转，而 LR 通常不保存在栈上。如果单纯覆盖栈上数据，可能根本碰不到返回地址，劫持失败。在 x86 中，虽返回地址在栈顶，但叶子函数栈帧极小，溢出时极易先破坏相邻栈变量或产生段错误（Segmentation Fault），导致劫持还没来得及发生程序就崩溃了。

所以我们在劫持程序执行流的时候一般选择非叶子函数。
